Class: App

Defined in: lib/cocoa-app.coffee

Overview

App class is the main Cocoa UI class. There is a specific way to import it in order to control when the $.import "AppKit" is started (see example). It is important to understand that run() will start the Cocoa runloop and node event loop is ignored afterwards. Therefor App creates an RPC server using the Runloop version of CocoaAsyncSocket Framework (https://github.com/robbiehanson/CocoaAsyncSocket). AppProxy will communicate with this RPC server

Examples:

App = require("cocoa-ui").App()

app = new App()
.onReady ->
  createUI()
.onStopping ->
  debug "Stopping"
.run()

# (void) openDialog(callback, options = {})

Open the system file open dialog as a separate window (non modal)

return [NSOpenPanel] the Cocoa dialog instance

Examples:

options =
  folder:"/Users/bernhard/Development"
  multipleSelection: true
  chooseFolders: true
  chooseFiles: true
  hiddenFiles: true
  title: "Test open"
  prompt: "Get it"
  message: "Select multiple files as you like"

app.openDialog((selection) ->
  console.log selection
, options)

Parameters:

  • callback ( function ) — The function to be called after file(s) have been selected or dialog has been cancelled

Options Hash: (options):

  • folder ( String ) — Starting folder
  • multipleSelection ( Boolean ) — Allow to select more than one file
  • chooseFolders ( Boolean ) — Allow to select folders (instead of opening them)
  • chooseFiles ( Boolean ) — Allow to select files
  • hiddenFiles ( Boolean ) — Show hidden files/folders
  • title ( String ) — Title of the dialog window
  • prompt ( String ) — Prompt shown in the open File Dialog
  • message ( String ) — Message shown in the open File Dialog

# (App) on(name, callback)

Register events to be called from AppProxy via rpc

Examples:

# in App()
app.on "aNewEvent", (params...) ->
   doSomething app, params ...

# in AppProxy
appProxy.rcall("aNewEvent")(param1, param2, ...)
.then (result) ->
  console.log "callback result", result

Parameters:

  • name ( String ) — Name of the event
  • callback ( Function ) — Callback function to be called when AppProxy calls the event via rpc

Returns:

  • ( App ) — for success to allow call chaining, else false (breaking the chain)

# (void) fireEvent(event, args...)

fireEvent allows the App to call handlers in the AppProxy. No return, it is fire and forget.

Examples:

# In the AppProxy
appProxy = new AppProxy()
.on "handleOk", (arg) ->
  debug "handleOk", arg

# in the App
app.fireEvent "handleOk", {"a":42, b:false}

Parameters:

  • event ( String ) — Name of the event
  • args... ( Array<Object> ) — Arguments for the rpc call

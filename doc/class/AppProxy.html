Class: AppProxy

Defined in: lib/app-proxy.coffee

Overview

AppProxy class is the counterpart of the App class. It starts an Rpc client and provides methods to

  • register handler for GUI events
  • remote procedure calls to access/modify GUI objects
  • add a AppProxy("window") and to this and AppProxy("control")

Examples:

{AppProxy} = require("cocoa-ui")

(app = new AppProxy())
.on("event", ->
  console.log("GUI fire an event")
)
.connect( ->
  console.log("Main logic here")
  app.rcall("guiHandler")(args...))

Instance Method Summary

  • # (AppProxy) window(tag) Subcomponent "window", that will link to Window object in the GUI App Allows to call remote methods of class Window
  • # (AppProxy) control(tag) Subcomponent "control" that will link to a UiControl derived GUI element in the GUI App.
  • # (AppProxy) connect(callback) Connect to RPC server of App class
  • # (AppProxy) on(name, callback) Register a callback that gets executed when a GUI control fires this event Allows to react on GUI changes in the AppProxy
  • # (AppProxy) handler(request) Bound Main handler to dispatch requests of form {method:"abc", params:args...} as abc(args...) for registered handlers (see "on")
  • # (Promise) rcall(method) Call a remote UI method which is either a GUI App method or a registered GUI App handler
  • # (void) try(func) Consistent error handling that close App process when a severe error happens in AppProxy.
